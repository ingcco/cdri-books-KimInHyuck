# 기획 명세 — certicos Books (테스트 기준 문서)

> **용도**: unit / integration / e2e 테스트의 기준(acceptance criteria). 각 요구사항에 **AC(WHEN→THEN) + 검증 레벨**을 붙여 테스트와 1:1 대응시킨다.
> **출처 태그**: **[F]** Figma 파란 밴드 명시 · **[N]** Notion 명시 · **[판단]** 미명시 → 지원자 판단 · **[추가]** 스펙 없이 지원자가 의도적으로 추가 구현.
> **레벨 태그**: `u`=unit(순수 함수/훅) · `i`=integration(MSW+RQ 훅) · `e`=e2e(Playwright). Figma 실측 px 정합은 별도 `e2e/*.spec.ts`(SOT: `.docs/design/screens.md`).

## 1. 검색창 [F]

- placeholder: 통합검색 **"검색어를 입력하세요"**(시안 렌더값 채택) · 상세검색 input **"검색어 입력"**(Figma 주석)
- **Enter 키로 검색 실행** — debounce 없음(타이핑 중 검색 안 함, 명시 실행)

| AC | WHEN → THEN | 레벨 |
|---|---|---|
| 1.1 | 검색어 입력 후 Enter → 검색 실행, 결과 렌더 | e |
| 1.2 | 검색 실행 → 검색어 기록 추가(최대 8, 중복 시 제거 후 최신순 맨 앞) | u |
| 1.3 | 재방문(재시작) → 검색 기록 유지 (`localStorage["search-history"]`) | u/e |
| 1.4 | 인풋 focus → 기록 리스트 노출; 항목 클릭=재검색; X=개별 삭제 | e |
| 1.5 | 빈/공백 검색어 제출 → 무시(검색·기록 추가 안 함) [판단] | u/e |
| 1.6 | 기록 리스트 열림 + ↑↓ → 하이라이트 이동, Enter=선택, Esc=닫기 [추가] | e |

## 2. 결과 리스트 [F]

| AC | WHEN → THEN | 레벨 |
|---|---|---|
| 2.1 | 검색 진행 → **페이지당 10개**, 스크롤 하단 도달 시 다음 페이지 로드(무한 스크롤, `meta.is_end`로 종료) | i/e |
| 2.2 | 구매하기 클릭 → **새 탭**으로 다음 책 `url` 이동(`rel="noopener noreferrer"`) | e |
| 2.3 | 하트 클릭 → 찜 토글 | e |
| 2.4 | 상세보기 클릭 → 아코디언 확장/축소 (**단일 열림** [판단]) | e |
| 2.5 | 아코디언 가격: **할인가 없으면(`sale_price === -1`) 할인가 미노출** | u/e |
| 2.6 | 카운트 "도서 검색 결과 총 N건" (숫자만 `primary`) | e |

> 무한 스크롤 구현: **react-virtual**(`useBookListVirtualizer` — 마지막 가상 아이템 도달 시 `fetchNextPage`). IntersectionObserver 미사용. [추가]

## 3. 상세 검색 [F]

- 원문(18:608): *"'상세 검색 버튼' 클릭시 버튼 하단에 팝업 노출 / 검색 조건 클릭시 셀렉트 옵션 노출 / '검색하기 버튼' 클릭시 팝업은 닫히며 입력한 조건으로 상세검색 진행"*

| AC | WHEN → THEN | 레벨 |
|---|---|---|
| 3.1 | 상세검색 버튼 클릭 → 버튼 하단 팝업 노출 | e |
| 3.2 | 검색 조건 셀렉트 옵션: **제목(`title`) / 저자명(`person`) / 출판사(`publisher`)** | e |
| 3.3 | 검색하기 클릭 → 팝업 닫힘 + 해당 조건으로 검색 | e |
| 3.4 | **상호배타**: 상세검색 진행 시 통합 입력 초기화 / 통합검색 진행 시 상세 조건 초기화 (`key` 리셋) | e |

## 4. 찜하기 [F]

- 원문(744:313): *"찜하기 버튼 눌렀을 때 시점의 데이터를 가지고 있기 때문에, API 결과와 데이터가 상이할 수 있음 / 브라우저 재시작 후에도 찜하기 목록 유지"*

| AC | WHEN → THEN | 레벨 |
|---|---|---|
| 4.1 | 하트 클릭 → **클릭 시점 도서 스냅샷 저장**(`FavoriteBook` DTO — 렌더 필드만 큐레이션 [판단]) | u |
| 4.2 | 재시작 후 찜 목록 유지 (`localStorage["favorites"]`) | u/e |
| 4.3 | 찜 페이지 **클라 페이지네이션 10개씩**(무한 스크롤, react-virtual 재사용) | e |
| 4.4 | 카운트 "찜한 책 총 N건" (숫자 `primary`) | e |

## 5. 카카오 API 매핑 [N]

`GET https://dapi.kakao.com/v3/search/book` (Authorization: `KakaoAK {VITE_KAKAO_REST_API_KEY}` — 클라이언트 직접 호출, §7-1)

| 파라미터 | 사용 |
|---|---|
| `query` | 검색어 |
| `target` | 통합: 미지정 / 상세: `title`·`person`·`publisher` |
| `page`/`size` | 무한 스크롤: size=10, page 증가, `meta.is_end`로 종료 |

| 응답 필드 | 화면 매핑 |
|---|---|
| `title`,`authors[]`,`thumbnail` | 제목·저자(", " 조인 [판단])·표지 |
| `price`/`sale_price` | 원가/할인가 (`sale_price === -1` → 미노출) |
| `contents` | 책 소개 | `url` | 구매하기 새 탭 | `isbn` | 아이템 key·찜 식별자 [판단] |

## 8. 계획했으나 의도적으로 제외 (근거 있는 미구현)

| 항목 | 사유 |
|---|---|
| **로딩 스켈레톤** | 카카오 API 응답이 **충분히 빨라 지연 체감 감소 실익이 없음** → 스켈레톤 미도입. 로딩은 "검색 중입니다…" 텍스트(`role="status"`) 최소 처리로 대체 |
| **반응형(모바일~데스크톱)** | 시안이 **1920 데스크톱 단일**, 반응형 실익 없음 → **1920 고정**으로 확정 |
| **SEO/서버 프리페치(Hydration)** | Vite CSR 전환으로 SSR 자체 불가 → 보류 |

## 9. 미구현/미노출 (주의 — 테스트 제외)

- **`sort`(정확도/최신)**: `BookListParams` 타입엔 존재하나 **정렬 UI 없음** → 미노출. 테스트 대상 아님(향후 도입 시 활성화).
