# 기획 명세 — certicos Books

> 출처: 노션 과제 문서 + Figma 주석(파란 밴드) 전사. **[F]** = Figma 명시, **[N]** = 노션 명시, **[판단]** = 미명시로 지원자 판단(유의사항: "명시되지 않은 요구사항은 지원자 개인의 판단에 따라 작업합니다").

## 0. 페이지 구성

| 라우트 | 화면 | Figma 프레임 |
|---|---|---|
| `/` | 도서 검색 | 도서 검색 초기/결과 화면, 전체 검색, 상세 검색 |
| `/favorites` | 내가 찜한 책 | 내가 찜한 책 초기/화면 |

Header GNB: 도서 검색 ↔ 내가 찜한 책 탭 전환, 활성 탭 언더라인.

## 1. 검색창 [F]

- placeholder: "검색어 입력" (시안 렌더: "검색어를 입력하세요")
- Enter 키로 검색 실행
- **검색 기록**: 최대 8개, 8개 초과 시 오래된 순으로 삭제, **브라우저 재시작 후에도 유지** → localStorage
- 기록 항목: 클릭 시 해당 검색어로 재검색 [판단], X 클릭 시 개별 삭제 [F 시안]

## 2. 결과 리스트 [F]

- 검색 진행 시 10개 결과 노출 (**페이지 당 10개 아이템**) — **무한 스크롤**
- 구매하기 클릭: **새 탭**으로 해당 책 상세페이지(다음 책 `url`) 이동
- 하트 아이콘 클릭: **찜하기 토글**
- 상세보기 클릭: **accordion** 확장/축소
- 아코디언 내 가격: **할인가가 없으면 미노출**
- 카운트: "도서 검색 결과 총 N건"

## 3. 상세 검색 [F]

- 상세검색 버튼 클릭 → 버튼 하단 팝업 노출
- 검색 조건 클릭 시 셀렉트 옵션 노출: **제목 / 저자명 / 출판사**
- 검색하기 클릭 → 팝업 닫히고 해당 조건으로 검색
- **전체 검색과 상세 검색은 동시 진행 불가**:
  - 상세 검색 시 기존 검색어 입력값 초기화
  - 상세 검색 도중 전체 검색 진행 시 상세 검색 조건 초기화

## 4. 찜하기 [F]

- **버튼 누른 시점의 데이터 스냅샷 저장** — "API 결과와 데이터가 상이할 수 있음" 명시 → 책 데이터 전체를 localStorage에 저장하는 것이 기획 의도
- **브라우저 재시작 후에도 찜 목록 유지** → localStorage
- 찜 페이지도 페이지 당 10개 아이템 [F] → 클라이언트 페이지네이션(무한 스크롤) [판단]
- 찜 목록 카운트: "찜한 책 총 N건"

## 5. 카카오 API 매핑 [N]

`GET https://dapi.kakao.com/v3/search/book` (Authorization: `KakaoAK {REST_API_KEY}` — **서버 전용**)

| 파라미터 | 사용 |
|---|---|
| `query` | 검색어 |
| `target` | 전체 검색: 미지정 / 상세 검색: `title`(제목)·`person`(저자명)·`publisher`(출판사) |
| `page` / `size` | 무한 스크롤: size=10, page 증가, `meta.is_end`로 종료 |

| 응답 필드 | 화면 매핑 |
|---|---|
| `title`, `authors[]`, `thumbnail` | 제목·저자(복수 시 ", " 조인 [판단])·표지 |
| `price` / `sale_price` | 원가 / 할인가 (sale_price === -1 → 미노출) |
| `contents` | 책 소개 (빈 값 처리 [판단]) |
| `url` | 구매하기 새 탭 이동 |
| `isbn` | 아이템 key·찜 식별자 [판단] |

## 6. 미명시 → 지원자 판단 목록 (근거 포함)

| # | 판단 | 근거 |
|---|---|---|
| 1 | 카카오 호출은 Next.js Route Handler 프록시 경유 | REST 키 브라우저 노출 방지 [N: API Key 보안] |
| 2 | 반응형(모바일~데스크톱) 대응 | 시안은 1920 데스크톱만. 원티드 우대사항(반응형) + 실서비스 표준 |
| 3 | 로딩 스켈레톤 + 에러 상태 UI | 시안 미정의. UX·CLS 방지 |
| 4 | 검색 상태 URL 동기화(nuqs): `?q=`, `?target=` | 공유 가능한 검색 URL, 새로고침 유지 — SEO·UX 어필 |
| 5 | 찜 식별자 = `isbn` | 카카오 응답의 유일 식별 후보 |
| 6 | 검색 기록 노출 = 인풋 focus 시 | 시안 흐름 (입력 중 화면) |
| 7 | 빈 검색어 제출 무시 | 방어적 처리 |
| 8 | 아코디언 다중 열림 허용 여부 → **단일 열림** | 시안이 항상 1개만 확장된 상태로 표현 |
| 9 | 무한 스크롤 = IntersectionObserver + `useInfiniteQuery` | RQ 필수 스택과 정합 |
| 10 | 접근성: 하트 `aria-pressed`, 팝업 포커스 트랩, 키보드 조작 | 원티드 우대(웹 접근성/웹 표준) |
| 11 | SEO: 메타데이터·OG, 서버 프리페치(Hydration) | 사용자 방침(prefetch SEO) |
| 12 | 책 소개 본문 10px는 시안 준수하되 최소 가독성 검토 | 시안 실측값 존중 + a11y 확인 |

## 7. 범위 제외 (하지 않는 것)

- 로그인/회원 기능, 서버 DB — 과제 범위 아님
- 구매 플로우 자체 구현 — 외부 링크 새 탭이 명세
- 페이지네이션 UI(번호) — 무한 스크롤이 명세
