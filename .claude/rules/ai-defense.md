# AI CLI 공통 이슈 방어

AI 코딩 어시스턴트 사용 시 발생하는 공통 문제에 대한 방어 규칙.

## 할루시네이션 방지

- 존재하지 않는 함수/API/파일 경로를 사용하기 전에 **반드시 Glob/Grep/Read로 확인**
- 라이브러리 API가 불확실하면 **WebFetch로 공식 문서 확인**(권위 도메인은 `external-docs.md`) 후 사용
- 확실하지 않은 정보는 "확인이 필요합니다"라고 명시 — 추측으로 진행 금지

## 데이터 스키마 날조 방지

- UI 구현 시 **카카오 도서 검색 API 응답 스펙**(`developers.kakao.com` 문서 + 실제 응답의 `documents`/`meta` shape)을 먼저 확인
- 응답에 정의되지 않은 필드를 타입/mock에 **임의 추가 금지** — 외부 레퍼런스(디자인 시안, 벤치마킹 등)에서 본 필드라도 API 응답에 없으면 사용하지 않음
- 응답에 없는 파생 값(할인율, 재고 상태 등)이 필요하면 **응답 필드로부터 계산 가능한지 먼저 확인**하고, 불가하면 "기획 협의 필요"로 명시
- mock 데이터(MSW 포함)는 **실제 API 응답 스펙과 1:1 매칭** — 편의를 위한 필드 확장 금지

## Context Drift 방지

- 긴 세션(1시간+)에서 규칙 무시가 발생하면 `/clear` 후 새 세션
- 관련 없는 작업을 같은 세션에서 이어하지 않기 — 작업 단위로 세션 분리
- 복잡한 조사는 subagent에 위임하여 메인 컨텍스트 보호

### Context Clear 타이밍 — 4가지 트리거

아래 중 하나라도 해당하면 사용자에게 `/clear`를 요청하고 재시작 프롬프트를 제공한다:

| 트리거                   | 조건                                                   |
| ------------------------ | ------------------------------------------------------ |
| **Planning → 구현 전환** | Phase 0~3 완료, plan doc 생성 직후 (항상)              |
| **Phase 완료**           | 3 Phase+ 또는 10파일+ 대규모 작업에서 각 Phase 완료 시 |
| **디버깅 반복**          | 같은 에러가 3회 이상 반복된 경우                       |
| **자동 압축 알림**       | Claude Code가 "이전 메시지 자동 압축" 알림 출력 시     |

**요청 포맷** (트리거 발생 시 사용자에게 출력):

```
📋 Context Clear 권장
지금 `/clear` 실행 후 아래 프롬프트로 새 세션을 시작해주세요:

"{plan doc 경로} 기준 Phase N+1 진행해줘. 완료된 Phase: 1~N"
```

plan doc는 항상 최신 상태로 유지되므로 새 세션은 문서만으로 컨텍스트를 완전 복구한다.

### `/clear` vs `/compact` 선택 가이드

| 명령       | 동작                                                | 권장 시점                                                                                                  |
| ---------- | --------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| `/clear`   | 컨텍스트 전부 폐기 + 새 세션                        | 작업 단위 전환(plan A → plan B), 디버깅 3회 실패, plan 생성 직후 구현 전환. 복구는 plan doc·메모리로        |
| `/compact` | 이전 메시지 자동 요약 + 핵심 보존                   | 동일 작업 장기 세션(1시간+) 지속 필요 + 초반 결정 사항을 잃으면 안 될 때. 토큰 절감 + 연속성 유지            |
| 자동 압축  | Claude Code가 임의 시점에 압축 (사용자 알림 후)     | 위 두 명령을 놓쳤을 때의 fallback. 알림 출력 시 `/clear` 검토                                              |

**원칙**: 같은 plan 문서 내 작업이면 `/compact`, 다른 plan/작업 전환이면 `/clear`. plan doc가 최신이면 `/clear` 후 새 세션도 doc만으로 복구.

## 파일 잘림 방지

- 긴 파일(200줄+) 전체를 재작성하지 않기 — **Edit 도구로 필요한 부분만 수정**
- Write 도구 사용 시 반드시 Read로 기존 내용을 먼저 확인
- 파일 수정 후 의도하지 않은 코드 삭제가 없는지 `git diff`로 확인

## Stale Knowledge 대응

- 라이브러리 버전이 불확실하면 `package.json`에서 실제 버전 확인
- deprecated API 사용 의심 시 공식 문서(권위 도메인)로 최신 문서 확인
- Stack Overflow 답변보다 **공식 문서** 우선

## 과도한 엔지니어링 방지

- 요청된 범위만 구현 — 추가 기능, 리팩토링, "개선" 자발적 수행 금지
- 환경/빌드 문제 시 **캐시 삭제(`.next`, `node_modules/.cache`)를 먼저** 시도 — config 추가/변경은 최후 수단
- 3곳 미만 사용 로직은 추상화하지 않고 인라인 유지
- "이것도 하면 좋겠다"는 제안은 구현하지 말고 **말로만 제안**

### 레퍼런스 프로젝트 구조 맹목적 복사 금지

`add-dir`로 연결한 다른 레포의 폴더 구조·네이밍을 참고할 때, 그 구조가 **왜** 그렇게 돼 있는지 먼저 확인한다 — 표면 패턴만 옮기면 그 구조의 전제가 없는 프로젝트에 안 맞는 흔적이 남는다.

- 예: `lib/api/client/`라는 이름은 실제 서버 폴더(`app/api/`)와 대응되는 Next.js 프로젝트에서만 의미가 있음. 서버가 없는 CSR 단독 앱에 그대로 가져오면 대응 대상이 없는 vestige가 됨
- 채택 전 자문: "이 구조가 존재하는 이유가 이 프로젝트에도 있는가?" — 없으면 이름/구조를 우리 상황에 맞게 조정

### 에이전트 위임 시 스코프 엄수

- 메인 → 서브에이전트 위임 시, **plan에 명시되지 않은 자발적 변경 금지**를 프롬프트에 반드시 포함
- 위임 프롬프트 의무 문구 (3종 모두 포함):
  - "스코프: {plan 항목 N개}만 처리. plan에 없는 변경은 전부 발견사항 보고로 회신"
  - "명시되지 않은 컨벤션 정정·리팩토링·raw HTML 교체·import 정리 금지"
  - "추가 작업이 필요해 보이면 backlog에 추가 후 종료 — 직접 수행 금지"
- 위반 발견 시 메인이 즉시 차단하고 backlog로 이관 — 작업 자체는 정당해도 plan 외 영역은 별 ticket으로 분리

## Silent Failure 방지

- 입력 검증·에러 처리 코드를 수정할 때 기존 보호 로직을 제거하지 않기
- 테스트를 통과시키기 위해 검증을 우회하지 않기
- 파일 수정 후 타입체크(`tsc --noEmit` 또는 프로젝트 typecheck) 결과 확인 — 에러 무시 금지

## 반복 실수 방지

- memory에 기록된 Gotcha를 작업 전 참조
- 같은 에러가 2회 이상 발생하면 근본 원인 분석 — 증상만 고치지 않기
- 해결한 이슈가 memory에 없으면 기록 제안
